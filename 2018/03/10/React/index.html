Hello React

通过这段时间以来，项目总算是在磕磕绊绊中完成了，经历了布局的困难，还有掉后台数据跨域的“火山”，更有路由的坑，更不能忘记的使用（Ant Design Mobile）插件的痛苦 sass

首先总结下React 的特点

(1)声明式设计
(2)高效率 （相比以前的项目开发，运用框架的话，销量确实大大的提高了）
(3)运用灵活 （这就不多讲了）
(4)JSX语法 （这个有了解但是我这次项目中，所以 略）
(5)组件化开发 （个人理解：把各个页面变成组件，里面的各个小局部也可以变成组件，想用哪里引入那里，方便）
(6)单项响应的数据流 （这个可以减少代码的重复）

布局

然后就是布局，布局到时没有什么困难，就是布局的时候，一定要全方位考虑问题，不要仅仅是布了一个静态页面的局，后期调用数据以后会发生什么变化，数据里面的字数如果溢出了，要不要添加省略号，还是要换行，图片大小调用出来的时候需要设置多大，这些都是需要考虑进去的问题，如果布局考虑全面会给自己还有同事提升不少的效率。（布局建议运用sass 提高效率，结构明确 杠杠的，运用sass记得插入中间件 sass使用： 安装node-sass sass-loader，在webpack的配置文件里面 添加sass相关的配置）

布局完成后就是路由跳转，路由的原理， 通过onhashchange事件去监听 hash值（”#/list”）改变， 然后渲染对应的组件(ListPage)。这个也没有太大的问题就是设计二级路由的时候，一定要注意把跟路由去掉，（ps：新手定义路由的时候，建议起名规范，要么注释清楚，页面多起来找路由你懂得….）

当路由搞定以后就可以掉用数据了，调用数据的时候，首先考虑的问题是，数据的类型以及跨域问题，这个一定要和后端沟通好

一、数据获取

(1) Jsonp ：使用fetch-jsonp插件
(2) 跨域问题：在package.json里配置服务器代理 proxy
(3)参数格式优化: qs模块(qs.stringify 可以 把 {a:1,b:2} =转换成> a=1&b=2)
(4)Fetch携带cookie ：credentials: “include”
(5)用户权限问题，需要复制原网站的cookie或者设置其他request headers

二、antd-mobile

  (1) 样式自动加载： 安装 babel-plugin-import插件，在webpack配置文件里babel-loader  options
(2)  自定义主题：  安装  less less-loader ，在package.json里面添加theme 设置；

在webpack 配置文件里， 加载theme设置，在loader里面添加less相关的配置
